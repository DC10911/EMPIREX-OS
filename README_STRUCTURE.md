# EMPIREX OS — Project Structure

מטרת המבנה: לעבוד מסודר תוך כדי תנועה, בלי בלגן קבצים.

## תיקיות ראשיות
- `01_Brand` — שם מותג, לוגו, שפה מותגית, נכסי זהות.
- `02_Design` — כל העיצוב (UI kit, wireframes, השראות, guidelines).
- `03_Content` — תוכן שיווקי/מוצרי, מחקר, פרומפטים, מדיה.
- `04_Product` — דרישות מערכת, ארכיטקטורה, roadmap.
- `05_Marketing` — דפי נחיתה, קמפיינים, סושיאל.
- `06_Operations` — אדמין, משפטי, פיננסי.
- `07_Development` — frontend/backend/database/infra.
- `99_Archive` — גרסאות ישנות/לא פעילות.

## חוק עבודה קצר
1. כל קובץ נכנס ישר לתיקייה המתאימה (לא לשורש).
2. שמות קבצים באנגלית, עם תאריך בתחילת קבצים חשובים.
3. גרסאות עיצוב: `v1`, `v2`, `final` רק אחרי אישור.
4. קבצי מקור + קבצי יצוא נשמרים בנפרד.

## תחילת עבודה מומלצת
- דף נחיתה: `05_Marketing/Landing_Page`
- מחקר וקופי: `03_Content/Research` + `03_Content/Copywriting`
- עיצוב ראשוני: `02_Design/Wireframes`
